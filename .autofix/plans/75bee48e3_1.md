# Issue 1: Documentation uses lowercase "share" instead of uppercase "SHARE"

## What and where
In `libs/mng/docs/customization.md`, line 182, the text says:
> If a non-symlink file or directory already exists at the target location when `"share"` mode would create a symlink, an error is raised.

The enum `WorkDirExtraPathMode` is case-sensitive and only accepts uppercase values `"SHARE"` and `"COPY"`. Pydantic validation rejects lowercase input like `"share"`.

## Why it is a problem
If a user reads this documentation and infers that lowercase is acceptable (since the docs reference `"share"`), they may write `"share"` in their `.mng/settings.toml` file. Pydantic will then reject it with a validation error, which is confusing.

## Planned fix
Change `"share"` to `"SHARE"` on line 182 of `libs/mng/docs/customization.md`.

## Risks
None -- this is a purely textual documentation fix.
