# Issue 2: Unrealistic mock data in tui_test.py

## What and where
In `libs/mng_kanpan/imbue/mng_kanpan/tui_test.py`, the test
`test_first_load_pr_failure_shows_prs_not_loaded` creates a `BoardSnapshot`
with `repo_pr_loaded={}` and a "gh pr list failed" error, but in production
a failed PR fetch would produce `repo_pr_loaded={"org/repo": False}`.

## Why it is a problem
Same reason as issue 1: the test validates correct behavior but with mock
data that doesn't match production. If `_classify_entry` is later changed
to distinguish "repo absent from dict" from "repo is False", this test
would pass while production behavior could be wrong.

## Planned fix
Change `repo_pr_loaded={}` to `repo_pr_loaded={"org/repo": False}` in the
test's `BoardSnapshot` construction.

## Risks
None -- the test validates the same rendering behavior with more realistic data.
