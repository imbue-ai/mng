# Fix: GitHubData.prs_loaded default should be False

## What the issue is and where

In `libs/mng_kanpan/imbue/mng_kanpan/data_types.py`, line 108, the `prs_loaded` field on `GitHubData` has a default value of `True`.

## Why it is a problem

With the new multi-repo design, `prs_loaded` means "whether PR data was successfully fetched for at least one repo." An empty `GitHubData()` constructed with no arguments has no loaded data, but `prs_loaded` defaults to `True`, which is semantically incorrect.

While current production code (`fetch_github_data`) always sets `prs_loaded` explicitly via `len(prs_loaded_repos) > 0`, and all test sites explicitly pass `prs_loaded=False` or `prs_loaded=True`, the incorrect default is a latent bug. Any future code that constructs `GitHubData()` without specifying `prs_loaded` would incorrectly report that PRs have been loaded.

## The planned fix

Change the default value of `prs_loaded` in `GitHubData` from `True` to `False`.

This is safe because:
1. `fetch_github_data` always sets it explicitly
2. All test construction sites already pass the value explicitly
3. No other code constructs `GitHubData` without specifying `prs_loaded`

## Risks or edge cases

None. All existing construction sites explicitly provide the value, so changing the default has no runtime effect on existing code.
