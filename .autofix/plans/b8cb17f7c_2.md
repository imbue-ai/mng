# Issue 2: Repetitive test setup in work_dir_extra_paths tests

## What and where
File: `libs/mng/imbue/mng/hosts/host_test.py`, lines 2202-2358
All 8 `test_apply_work_dir_extra_paths_*` functions.

## Why it is a problem
Each of the 8 test functions repeats the same pattern:
```python
source_dir = tmp_path / "source"
source_dir.mkdir()
work_dir = tmp_path / "work"
work_dir.mkdir()
```
This is duplicated setup code that should be factored out into a fixture.

## Planned fix
Add a fixture to `libs/mng/imbue/mng/hosts/conftest.py` that provides `source_dir` and `work_dir` as a tuple. Then update all 8 tests to use it instead of repeating the setup.

The fixture will:
1. Create a `source` subdirectory in `tmp_path`
2. Create a `work` subdirectory in `tmp_path`
3. Return both as a tuple `(source_dir, work_dir)`

Each test will unpack the fixture and use the directories directly.

## Risks
- Need to verify that conftest.py is the right place (per CLAUDE.md, all fixtures must be in conftest.py)
- Need to make sure the fixture uses `tmp_path` which gives each test its own temp directory
