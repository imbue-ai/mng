# Issue 1: Duplicate agent-finding logic in transcript.py

## What the issue is

`_find_agent_by_identifier` in `libs/mngr/imbue/mngr/api/transcript.py` duplicates functionality that already exists in `libs/mngr/imbue/mngr/api/find.py` as `resolve_agent_reference`.

## Where it is

- `libs/mngr/imbue/mngr/api/transcript.py`, lines 82-92 (the `_find_agent_by_identifier` function)
- The call site at line 51

## Why it is a problem

1. Code duplication: The codebase already has `resolve_agent_reference` in `find.py` that does the same thing (finds an agent by name or ID across all hosts).
2. Missing edge case handling: The duplicate function does not handle the case where multiple agents match the same name -- it silently returns the first match. The existing `resolve_agent_reference` properly detects ambiguous matches and raises a clear error.
3. Different error types: The duplicate raises `AgentNotFoundError` while the existing raises `UserInputError`. The existing behavior is more appropriate since this is user input validation.

## Planned fix

1. Remove `_find_agent_by_identifier` from `transcript.py`
2. Import and use `resolve_agent_reference` from `find.py` instead
3. Update the call site in `get_agent_transcript` to use `resolve_agent_reference` with appropriate parameters
4. Update the test file to remove tests for the deleted function and adjust any tests that depend on the specific error type
5. Note: `resolve_agent_reference` returns `None` when identifier is `None`, but since transcript requires an agent, we can pass the identifier and handle the None case (which won't occur since the identifier is required).

## Risks and edge cases

- `resolve_agent_reference` raises `UserInputError` instead of `AgentNotFoundError` for the not-found case. This changes the error type but `UserInputError` is more appropriate for a CLI-facing error.
- `resolve_agent_reference` takes an optional `resolved_host` parameter -- we'll pass `None` since transcript searches all hosts.
- Need to make sure the test importing `_find_agent_by_identifier` is updated.
