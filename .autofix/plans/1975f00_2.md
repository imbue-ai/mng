# Issue 2: Unescaped target_str in shell double-quoted string

## What and Where
In `libs/mng/imbue/mng/hosts/host.py`, line 1526, `target_str` is interpolated directly
into a shell double-quoted string in the CONFLICT error message accumulation:

```python
f'errors="${{errors}}CONFLICT: {target_str}\n"; '
```

## Why it is a problem
Inside a shell double-quoted string, characters like `$`, backticks, and `\` have special
meaning. If `target_str` contains any of these (e.g., a path like `/home/user/$project/`),
the shell would try to expand them. This would cause:
1. The CONFLICT error message to contain incorrect path text
2. The conflict detection in Python (lines 1535-1538) to fail to match the original path
3. Potential unintended command execution if the path contains backticks
