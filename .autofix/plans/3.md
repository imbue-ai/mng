# Issue 3: session_file_path uses `str` instead of `Path` for file path

## What the issue is

In `libs/mngr/imbue/mngr/api/transcript.py`, the `TranscriptResult.session_file_path` field is typed as `str` (line 22), and `_find_session_file` returns `str` (line 91). The style guide requires: "Always use pathlib.Path instead of str for file paths and directory paths."

## Where it is

- `libs/mngr/imbue/mngr/api/transcript.py`, line 22: `session_file_path: str`
- `libs/mngr/imbue/mngr/api/transcript.py`, line 91: return type of `_find_session_file` is `str`
- `libs/mngr/imbue/mngr/api/transcript.py`, line 78: `Path(session_file_path)` conversion
- `libs/mngr/imbue/mngr/cli/transcript.py`, line 93: `result.session_file_path` used in dict
- Test files that reference `session_file_path` as string values

## Why it is a problem

The style guide is explicit: "Always use pathlib.Path instead of str for file paths and directory paths." The codebase consistently uses `Path` for paths, even remote host paths (e.g., `HostLocation.path` is `Path`). Using `str` here is inconsistent and violates the style guide.

## Planned fix

1. Change `session_file_path: str` to `session_file_path: Path` in `TranscriptResult`
2. Change `_find_session_file` return type to `Path` and wrap the return value in `Path()`
3. Remove the `Path()` conversion at the `read_text_file` call site (line 78) since the value will already be a `Path`
4. Update `_emit_json_output` in cli/transcript.py to convert `Path` to `str` for JSON serialization
5. Update test assertions to use `Path` objects where appropriate, or use `str()` conversion in comparisons

## Risks and edge cases

- The JSON output dict needs `str(result.session_file_path)` since JSON doesn't natively support Path objects.
- Test assertions using string comparisons on `session_file_path` need updating.
- The `session_file_path` comes from stdout of a remote `find` command, so it starts as a string. Wrapping in `Path` at the boundary (in `_find_session_file`) is the right approach.
