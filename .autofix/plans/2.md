# Issue 2: Add integration test for --format-template

## What the issue is and where

The integration test file `libs/mngr/imbue/mngr/cli/test_list.py` does not include any tests for the new `--format-template` feature. Every other output mode and major option has integration test coverage.

## Why it is a problem

The `--format-template` feature is a significant new capability. Without integration tests, we cannot verify that the full end-to-end path works correctly (CLI parsing -> template expansion -> output). The unit tests only cover the inner `_render_format_template` and `_emit_template_output` functions in isolation.

## The planned fix

Add two integration tests to `libs/mngr/imbue/mngr/cli/test_list.py`:

1. `test_list_command_format_template_with_agent` -- Create an agent, then list with `--format-template "{name}\t{state}"` and verify the output contains the expected template-expanded text.

2. `test_list_command_format_template_no_agents` -- Run `--format-template` with no agents and verify empty output (no "No agents found" message, since template mode is for scripting).

## Risks or edge cases

- The test relies on the CLI runner capturing stdout output, which should work since `_emit_template_output` and `_list_streaming_template` both write to `sys.stdout`.
- Need to ensure the CliRunner does not use catch_exceptions=True to get proper error reporting.
