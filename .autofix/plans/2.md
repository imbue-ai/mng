# Issue 2: _build_updated_permissions uses list instead of Sequence for input

## What and Where
In `libs/mngr/imbue/mngr/cli/limit.py` line 133, `_build_updated_permissions` takes `current: list[Permission]` as a parameter.

## Why It's a Problem
The style guide requires function inputs to use immutable abstract types: `Sequence[T]` instead of `list[T]`. This ensures the type checker can catch accidental mutations of input data.

## Planned Fix
Change `current: list[Permission]` to `current: Sequence[Permission]` and add the necessary import for `Sequence` from `collections.abc`.

## Risks
None -- `Sequence` is a supertype of `list`, so all existing call sites remain valid.
