# Issue 2: _find_session_file uses `object` type instead of `UUID` for agent_uuid parameter

## What the issue is

In `libs/mngr/imbue/mngr/api/transcript.py`, the `_find_session_file` function has parameter `agent_uuid: object` (line 89). The value passed to this parameter comes from `matched_agent_ref.agent_id.get_uuid()` which returns `UUID` from the `uuid` module.

## Where it is

- `libs/mngr/imbue/mngr/api/transcript.py`, line 89: `agent_uuid: object`

## Why it is a problem

The style guide emphasizes using the type system to constrain code and provide assurance of correctness: "Use the power of the type system to constrain your code and provide some assurance of correctness." Using `object` as the type annotation provides no type safety -- any value could be passed. The parameter should be typed as `UUID` to match the actual data being passed.

## Planned fix

1. Add `from uuid import UUID` to the imports in `transcript.py`
2. Change `agent_uuid: object` to `agent_uuid: UUID` in `_find_session_file`

## Risks and edge cases

- None. The parameter is only called with `UUID` values from `get_uuid()`, so tightening the type is safe.
