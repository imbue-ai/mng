# Issue 1: Documentation uses lowercase mode names but only uppercase is accepted

## What and where
In `libs/mng/docs/customization.md`, lines 169-170, the mode descriptions say:
- `"share"`: On the same host, creates a symlink...
- `"copy"`: Always copies the path via rsync...

But `WorkDirExtraPathMode` (in `data_types.py`) is an `UpperCaseStrEnum` whose values are `"SHARE"` and `"COPY"`. Pydantic validation rejects lowercase input.

## Why it is a problem
A user reading the documentation would see the lowercase mode names in the bullet-point descriptions and might use them in their config. This would cause a confusing Pydantic validation error at config load time. The TOML example above correctly uses uppercase, creating an inconsistency within the same documentation section.

## Planned fix
Change the mode descriptions in the Modes section from lowercase to uppercase:
- `"share"` -> `"SHARE"`
- `"copy"` -> `"COPY"`

Also update the "When to use each mode" section which references lowercase `"share"` and `"copy"`.

## Risks
None -- this is a documentation-only change.
