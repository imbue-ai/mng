# Issue 1: @pure decorator on _check_local_symlink_state

## What and Where
`_check_local_symlink_state` at line 97 of `libs/mng/imbue/mng/hosts/host.py` is decorated with `@pure`.

## Why It Is a Problem
The `@pure` decorator is advisory documentation indicating a function:
- Has no side effects
- Does not modify any state outside its scope
- Does not perform I/O operations
- Returns the same output for the same inputs

`_check_local_symlink_state` calls `target_abs.is_symlink()`, `target_abs.resolve()`, `source_abs.resolve()`, and `target_abs.exists()`, all of which perform filesystem I/O. The function's return value depends on filesystem state, not just its inputs. The decorator is therefore misleading.

## Planned Fix
Remove the `@pure` decorator from `_check_local_symlink_state`.

## Risks
None. The `@pure` decorator is advisory only (no runtime enforcement). Removing it just corrects the documentation.
