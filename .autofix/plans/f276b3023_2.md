# Issue 2: Missing error checking on symlink creation commands

## What the issue is

In `_apply_work_dir_extra_paths` at lines 1544-1545 of `libs/mng/imbue/mng/hosts/host.py`, the results of `execute_command` for `mkdir -p` and `ln -snf` are not checked for success.

## Why it is a problem

If the symlink creation fails (e.g., permission denied, disk full, or any other OS error), the failure is silently ignored. The code logs a debug message saying the symlink was created, but it may not have been. The user gets no indication that their "share" path was not actually linked into the work directory.

While the codebase has other unchecked `mkdir -p` calls (e.g., `_mkdir` at line 632), the symlink creation is different because:
1. It's user-facing config that the user explicitly set up
2. A silent failure means the work directory is subtly broken

## Planned fix

Check the result of both `mkdir -p` and `ln -snf` commands and raise `MngError` if either fails.

## Risks and edge cases

- This follows the pattern used elsewhere in the file (e.g., `git init --bare` at line 1222 checks result.success).
- The error messages should be actionable so the user knows which path failed.
