# Issue 1: Unrealistic mock data in fetcher_test.py

## What and where
In `libs/mng_kanpan/imbue/mng_kanpan/fetcher_test.py`, the test
`test_fetch_board_snapshot_surfaces_gh_errors_and_suppresses_create_pr_url`
creates a mocked `GitHubData` with `repo_pr_loaded={}` and an error string,
but the real `fetch_github_data` function always sets
`repo_pr_loaded[repo_path] = False` when a fetch fails.

## Why it is a problem
The test validates correct behavior (create_pr_url is suppressed on failure)
but uses mock data that doesn't match what production code produces. If
someone later changes `_classify_entry` to distinguish between "repo absent
from dict" and "repo is False in dict", this test could silently pass while
production behavior is wrong.

## Planned fix
Change `repo_pr_loaded={}` to `repo_pr_loaded={"org/repo": False}` in the
mocked `GitHubData`, and update the corresponding assertion from
`assert snapshot.repo_pr_loaded == {}` to
`assert snapshot.repo_pr_loaded == {"org/repo": False}`.

## Risks
None -- the test assertions still validate the same behavior (create_pr_url
is suppressed), just with more realistic mock data.
